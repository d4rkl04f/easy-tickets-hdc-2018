Reliability, Scalability, Maintainability

Reliability

  • Systems work correctly, even when things go wrong
  • Fault Tolerance
    • Hardware Errors
    • Software Errors (Application Code!)
    • Human Errors

Scalability

  • Systems designed to cope with increased load
  • Describing load
    • Writes vs. Reads
    • Twitter example
  • Describing performance
    • Percentiles
    • Amazon, P999 example

Maintainability

  • Easy to run "smoothly" (Operatbility)
  • Remove as much complexity as possible (Simplicity)
  • Conducive to future innovation (Evolveability)
